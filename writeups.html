🌦 Weather Prediction System (Machine Learning)

Machine Learning Data Science Web Application

Team Size: 4

Built a machine learning–based weather prediction system that forecasts rainfall occurrence (Yes / No) using 10 years of historical weather data. The system uses temperature, humidity, and pressure to deliver fast, data-driven predictions without complex meteorological models.

📌 Problem Statement

  • Traditional systems require high computational power
  • Depend on satellite and real-time sensor data
  • Complex and costly for small-scale applications

This project provides a lightweight, efficient alternative using supervised machine learning.

⚙️ How the System Works

  1. Data Collection: Historical weather data from NOAA, Kaggle, and OpenWeatherMap (10 years)
  2. Data Preprocessing: Handling missing values, normalization, and categorical encoding
  3. Feature Selection: Temperature, Humidity, Pressure
  4. Model Training: Logistic Regression (80% training, 20% testing)
  5. Model Evaluation: Accuracy, Precision, Recall, F1-Score
  6. Prediction Interface: User inputs parameters → instant rainfall prediction

🛠 Tech Stack

Python, Pandas, NumPy, Scikit-learn, Flask, HTML, CSS

🧪 Custom Privilege Escalation CTF Machine (Self-Created)

Linux Privilege Escalation CTF Lab Design Offensive Security

Project Type: Self-Designed Vulnerable Machine

Difficulty: Beginner → Advanced PrivEsc

Designed and built a custom Linux-based Capture The Flag (CTF) machine focused on post-exploitation and privilege escalation techniques. The system was intentionally misconfigured to simulate real-world Linux security flaws, allowing testers to practice enumeration, exploitation, and root compromise in a safe lab environment.

🎯 Learning Objective

  • Understand real-world Linux privilege escalation vectors
  • Practice deep enumeration and misconfiguration discovery
  • Demonstrate attacker mindset after initial access

⚙️ Lab Design & Enumeration

  • Hidden flags across user and system directories
  • Script analysis for embedded secrets
  • Recursive directory traversal and permission analysis

🚨 Privilege Escalation Techniques Implemented

  • Sudo Misconfiguration: NOPASSWD access to find (GTFOBins)
  • SUID Binary Abuse: Custom root-executing binary
  • Linux Capabilities Abuse: cap_setuid privilege escalation
  • Vulnerable Cron Job: Tar wildcard injection
  • World-Writable systemd Service: Root service manipulation
  • World-Writable Root Script: Arbitrary command execution as root

🏁 Flags & Validation

  • User-level flags (easy + hidden)
  • Root-level flags confirming full system compromise

🛠 Tools & Techniques

Linux, Bash, GTFOBins, File Permission Analysis, SUID, Capabilities, Cron, systemd

🛡 Defensive Takeaways

Provided remediation guidance including sudo hardening, permission fixes, removal of unnecessary SUID/capabilities, and enforcing least privilege.
